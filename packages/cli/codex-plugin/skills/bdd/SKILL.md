# BDD

Use Safe Word's behavior-first workflow: define observable scenarios first, then implement one scenario at a time with RED, GREEN, and REFACTOR evidence.

For full local project rules, inspect the active Safe Word ticket and its `test-definitions.md` ledger before changing code.

Example invocation: `safeword:bdd`.
