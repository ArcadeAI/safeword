One command. Your agent writes better code.

## Install

```bash
bunx safeword@latest setup
```

Takes about 30 seconds. Works with TypeScript, Python (beta), Go (beta), Rust (beta), and SQL/dbt (beta) projects.

Safeword adds code quality tools as devDependencies and configuration files. Your application code, runtime dependencies, and existing configs are unchanged.

## What Changes

You use Claude Code, Cursor, or Codex the same way you always have. Type a prompt, get code. But now there's a safety net running in the background.

<Tabs>
  <TabItem label="Claude Code">
    **Every edit gets linted automatically.** After your agent writes or modifies a file, Safeword runs your linter. If there are issues, your agent sees them immediately and fixes them—before you even notice.

    **Every session ends with a quality check.** When you're done working, Safeword prompts your agent to review what it built. Catches things like missing error handling, security issues, or code that doesn't match the codebase style.

  </TabItem>
  <TabItem label="Cursor">
    **Your agent follows structured workflows.** Safeword adds rules (`.cursor/rules/`) that guide how your agent approaches tasks. When debugging, it investigates before jumping to fixes. When building features, it follows your codebase patterns.

    **Better decisions, automatically.** The rules kick in based on what you're asking for—no special commands needed.

  </TabItem>
  <TabItem label="Codex">
    **Your agent gets scoped safeword plugin skills.** The Safe Word Codex plugin exposes names such as `safeword:bdd`, `safeword:verify`, and `safeword:explain` instead of installing Safe Word workflow files into your repo.

    **PreToolUse gates run through the plugin.** Run `safeword codex install` to install and verify the profile-scoped plugin, start a new Codex session to load it, then review and trust it in Codex. Its hooks use version-pinned Bunx commands. Safeword guards the documented Codex PreToolUse paths it configures: `Bash`, `apply_patch` edit payloads, and file-editing tools.

    **Stop nudges run through Codex continuations.** The packaged `safeword hook codex stop` entrypoint can continue the turn with advisory text after the agent stops. That is how done-phase reminders such as ARCHITECTURE.md drift nudges reach Codex; it is not hard done-gate enforcement.

  </TabItem>
</Tabs>

## Try It

Open your project and ask your agent to build something real:

> "Add an API endpoint that returns user profile data"

<Tabs>
  <TabItem label="Claude Code">
    Watch what happens:

    1. Your agent writes the endpoint
    2. Lint runs automatically on the new code, auto-fixing what it can
    3. If there are issues, your agent fixes them before you even see the code
    4. When you stop the session, a quality review runs automatically

  </TabItem>
  <TabItem label="Cursor">
    Watch what happens:

    1. Your agent investigates your existing API patterns first
    2. It follows the structure of your codebase
    3. If you ask it to debug something, it'll investigate before jumping to fixes

    The rules in `.cursor/rules/` guide this behavior automatically.

  </TabItem>
  <TabItem label="Codex">
    Watch what happens:

    1. Run `safeword codex install`, then start a new Codex session
    2. Review and trust the Safe Word plugin in Codex
    3. Your agent follows scoped plugin skills such as `safeword:bdd`
    4. Before supported edits, Codex runs the plugin's version-pinned Bunx PreToolUse hook
    5. After trust, run `safeword codex migrate --remove-legacy-hooks` to remove only Safe Word-owned legacy hooks
    6. If a safeword gate blocks the edit, your agent gets the denial before changing files
    7. When the agent stops, Codex runs the plugin's Bunx Stop hook for continuation nudges such as ARCHITECTURE.md drift reminders

  </TabItem>
</Tabs>

## What Got Installed

Safeword added a few folders to your project:

| Folder                  | What's in it                                             |
| ----------------------- | -------------------------------------------------------- |
| `.safeword/hooks/`      | Scripts that run automatically (lint, quality checks)    |
| `.claude/skills/`       | Structured workflows for debugging, BDD/TDD, refactoring |
| `.claude/settings.json` | Hook configuration for Claude Code                       |
| Safe Word Codex plugin  | Profile-scoped Codex workflow skills and hooks           |
| `.cursor/hooks.json`    | Hook configuration for Cursor                            |
| `.cursor/rules/`        | Rules for Cursor                                         |
| `.cursor/commands/`     | Slash commands for Cursor                                |
| `.safeword/`            | Config files, guides, and templates                      |

These project files commit to your repo. The Safe Word Codex plugin is profile-scoped: each Codex user runs `safeword codex install` and reviews it in `/hooks`; no Safe Word Codex workflow files are added to the repository.

For JS/TS projects, safeword also adds ESLint and Prettier as **devDependencies** in your `package.json`. These are development-only code quality tools — they don't ship with your application.

## Staying Current

Safeword improves over time. Update when you want the latest:

```bash
bunx safeword@latest upgrade
```

Your customizations are preserved. Only the core files get updated.
