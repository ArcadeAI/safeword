# Claude Code Changelog: Feb-Apr 2026

Covers: Hook changes, permission updates, agent workflow improvements, autonomy features, plugin system, and performance fixes from Claude Code v2.1.80-2.1.101.

## Hook-Relevant Changes

- **`"defer"` permission decision for PreToolUse hooks** (2.1.89) — new option beyond allow/deny/ask. Defers to the next hook or default behavior.
- **`PermissionDenied` hook event** (2.1.89) — fires after auto mode classifier denies an action. Can be used to log or react to denials.
- **`TaskCreated` hook event** (2.1.84) — fires when a task is created. Documented in 2.1.89.
- **`CwdChanged` and `FileChanged` hook events** (2.1.83) — fires when working directory changes or a file changes outside of Claude's tools.
- **`WorktreeCreate` hook for HTTP type** (2.1.84) — supports `worktreePath` in the hook input.
- **Conditional `if` field for hooks** (2.1.85) — hooks can use permission rule syntax to conditionally fire.
- **`hookSpecificOutput.sessionTitle` in UserPromptSubmit** (2.1.94) — hooks can read the session title.
- **PreToolUse hooks can satisfy `AskUserQuestion` with `updatedInput`** (2.1.85) — hooks can answer user questions programmatically.
- **Settings resilience: unrecognized hook event names no longer break** (2.1.101) — safer hook development.
- **prompt-type Stop/SubagentStop hooks fixed on long sessions** (2.1.98, 2.1.97, 2.1.92) — multiple fixes for prompt hooks failing in long sessions.
- **Hook `if` not matching compound commands fixed** (2.1.89) — conditional hooks now work with piped/chained commands.
- **PreToolUse/PostToolUse hooks `file_path` now absolute** (2.1.89) — previously relative.
- **Hook output >50K saved to disk** (2.1.89) — large hook output no longer truncated.
- **`permissions.deny` rules now properly override PreToolUse hook's `permissionDecision: "ask"`** (2.1.101) — deny takes precedence.

## Agent Workflow Changes

- **Named subagents in `@` mention typeahead** (2.1.89) — agents can reference each other by name.
- **Agents can declare `initialPrompt` in frontmatter** (2.1.83) — agents auto-start with a prompt.
- **Background subagents fixed: invisible after compaction, stuck running, not reporting progress** (2.1.83, 2.1.89, 2.1.101) — multiple reliability fixes.
- **Agent team members inherit leader's permission mode** (2.1.98) — permissions cascade.
- **Subagents no longer leak working directory** (2.1.97) — isolation improved.

## Autonomy & Effort Changes

- **Default effort changed from medium to high** for API-key, Bedrock/Vertex/Foundry, Team, Enterprise users (2.1.94) — agents think harder by default.
- **Auto mode classifier improvements** (2.1.89, 2.1.90, 2.1.101) — including denied command notifications, fixed boundary respect, denied commands in /permissions Recent.
- **Thinking summaries not generated by default** (2.1.89) — reduces overhead.
- **`/effort max` fixed for unknown model IDs** (2.1.98) — previously denied.

## Skills & Plugins

- **`disableSkillShellExecution` setting** (2.1.91) — disable inline shell execution in skills.
- **`effort` frontmatter for skills/commands** (2.1.80) — skills can set their own effort level.
- **Skill descriptions capped at 250 characters** (2.1.86) — enforces brevity.
- **`keep-coding-instructions` frontmatter for plugin output** (2.1.94) — controls how plugin output is displayed.
- **`/reload-plugins` picks up skills without restart** (2.1.98) — hot-reload.

## Performance

- **Fixed memory leak where long sessions retained historical message copies** (2.1.101) — significant for long sessions.
- **Fixed nested CLAUDE.md re-injected dozens of times** (2.1.89) — context pollution fix.
- **Autocompact thrash loop circuit breaker (3 attempts)** (2.1.89) — prevents infinite compaction.
- **Memory index truncates at 25KB** (2.1.83) — caps memory file size.
- **Non-streaming token cap increased 21k → 64k** (2.1.83) — larger outputs.
- **Edit tool uses shorter `old_string` anchors** (2.1.91) — reduces token usage.

## Permissions

- **`managed-settings.d/` drop-in directory** (2.1.83) — organizations can layer settings.
- **`forceRemoteSettingsRefresh` for fail-closed behavior** (2.1.92) — enterprise governance.
- **Many Bash permission hardening fixes** (2.1.89-2.1.101) — backslash escapes, compound commands, piped cd, env-var prefixes.
