# Add an NTB front door — "driving safeword without reading code"

**Goal:** Give the NTB a place that explains, in plain language, what they'll see and how to respond.

**Why (audit M2):** The README is organized for the Technical Builder and Maintainer (Development, CLI Parity, Publishing). Nothing addresses the NTB — the largest, highest-value audience per personas.md — directly. They need a short orientation: the agent will sometimes get stopped (that's safeword protecting you from shipping something broken), here's how to read a block, and type `/explain` when a message doesn't make sense.

## Scope sketch

- A short "Driving safeword without reading code" section in README (or a printed orientation after `setup`, coordinated with KRUEWC).
- Names the three things an NTB sees: blocks, verdicts, `/explain` — in plain words.
- Out of scope: restructuring the rest of the README.
