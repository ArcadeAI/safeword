# /audit pollutes working tree by writing depcruise configs

**Goal:** Make `/audit` a pure read-and-report surface. Stop the unconditional `sync-config` write at audit step 0; surface architectural drift as a `[W007]` warning instead so the customer chooses when to refresh and commit.

**Why:** A customer running `/audit` on a clean tree was left with `.dependency-cruiser.cjs` and `.safeword/depcruise-config.cjs` unstaged. Root cause: step 0 of the audit skill unconditionally runs `bunx safeword@latest sync-config`, which `writeFileSync`s both files. Even when content matches, a review skill mutating the tree muddies the customer's mental model — after the audit, `git status` no longer means anything reliable. The fix decouples drift _detection_ (audit's job) from drift _fixing_ (sync-config's job).

## Implementation sketch

1. Extract a `diffConfig(cwd, arch): { matches: boolean; diff?: string }` helper from `syncConfigCore` in `packages/cli/src/commands/sync-config.ts`. Reads existing `.safeword/depcruise-config.cjs` (if any), generates new content, byte-compares.
2. Add a `--check` boolean flag in the cli.ts commander wiring; pass through to `syncConfig()`.
3. When `--check` is set: call `diffConfig`, print "Config in sync" + exit 0 on match, or print a short unified diff (or just "Config drifted: <N> lines changed — run \`safeword sync-config\`") + exit 1 on mismatch. No writes either way. Missing on-disk file is treated as drift.
4. Replace the audit-skill bash line. Use exit code to set a shell variable, then conditionally emit the `[W007]` line in the audit's findings output. Update the warning legend.
5. Sync `.claude/skills/audit/SKILL.md` → `packages/cli/templates/skills/audit/SKILL.md` → `.cursor/commands/audit.md`. Per `feedback_template_sync.md`: hook tests run against templates/, not `.safeword/hooks/`; same parity discipline applies to skill files.
6. Tests: three new cases in `sync-config.test.ts` — match, drift, missing. Plus assert no writes happen under `--check` (snapshot `.safeword/depcruise-config.cjs` mtime / contents).
