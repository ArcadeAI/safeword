# Codex Hook Parity Map

The packaged command owns hook entrypoints; project files remain state and
customer content. Each preserved row has a deterministic subprocess proof.

| Legacy behavior | Decision | Rationale | Deterministic proof / follow-up |
| --- | --- | --- | --- |
| PreToolUse quality gate and proof identity | Preserve | The package command calls the same gate semantics and writes the proof identity bridge. | cucumber-js PreToolUse scenarios |
| PostToolUse quality state and skill nudge | Preserve | Package execution delegates to the shared Codex observers before any queued compatibility context. | cucumber-js PostToolUse scenarios |
| Stop architecture, retro extraction, and filing | Preserve | The package command delegates to `codex/stop.ts`; architecture retains precedence over filing. | cucumber-js Stop scenarios |
| SessionStart auto-upgrade and context | Preserve | One package-owned dispatcher runs shared upgrade logic, then package-owned SAFEWORD context. | cucumber-js SessionStart scenarios |
| UserPromptSubmit queued context | Preserve | The queue is project-owned data and remains silent when empty. | cucumber-js UserPromptSubmit scenarios |
| Plugin hook command rendering | Preserve | Every plugin event calls the public packaged command, never a repo-local adapter. | cucumber-js manifest scenario |
| Hidden `codex-hook` command alias | Preserve | Existing installations retain the previous CLI spelling while using the same implementation. | cucumber-js compatibility-alias scenario |
| Adapter-local crash capture | Defer | Package process-level attribution is retained; adapter-local capture needs a dedicated attribution design rather than a second crash boundary. | follow-up: assess only if package-level reports lack event attribution; explicit non-goal for this ticket |
